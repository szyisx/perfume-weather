# PerfumeWeather — Полное руководство для защиты

Этот документ объясняет проект **простым языком**, без глубокого Python.
Прочитав его, можно понять, как устроен сайт, и уверенно защитить работу перед преподавателем.

---

## 1. О чём проект

**PerfumeWeather** — сайт, который советует, какой парфюм надеть **по текущей погоде в вашем городе**.

**Идея простая:**
- Жара? Свежие цитрусовые ароматы.
- Холод и снег? Тёплые восточные с ванилью и удом.
- Дождь? Древесные ноты.

Юзер вводит город → сайт спрашивает погоду у внешнего сервиса → выдаёт подходящие парфюмы из своей базы.

## 4. Что такое Django (в двух словах)

Django — **готовая система**, в которой сайт построен из стандартных блоков:

| Блок | Что делает | Пример из проекта |
|------|------------|-------------------|
| **Модель (model)** | Описывает, какие таблицы в БД | `Perfume` — поля: имя, бренд, год, описание |
| **Представление (view)** | Обрабатывает HTTP-запрос и возвращает ответ | `PerfumeListView` — отдаёт каталог |
| **Шаблон (template)** | HTML с подстановкой данных | `perfume_list.html` — рисует карточки |
| **URL** | Маршрут: какой URL → какое представление | `/perfumes/` → `PerfumeListView` |
| **Форма (form)** | Описывает поля для ввода юзером | `PerfumeForm` — создание парфюма |
| **Миграция** | Файл-инструкция, как изменить БД | `0001_initial.py` создал все таблицы |

**Схема обработки запроса:**

```
браузер
  ↓ /perfumes/
URL-маршрут (urls.py)
  ↓ направляет на
View (views.py)
  ↓ берёт данные из
Модель ⇄ База данных
  ↓ передаёт в
Шаблон (.html)
  ↓ возвращает HTML
браузер
```

## 6. База данных — что хранится

В БД 10 таблиц:

| Таблица | Что хранит | Пример строки |
|---------|------------|---------------|
| `User` | Пользователи | admin, testuser |
| `Country` | Страны | Франция, Италия, США, ОАЭ |
| `Brand` | Бренды | Maison Aurea (Франция) |
| `Family` | Семейства ароматов | Цитрусовые, Восточные, Древесные |
| `Note` | Ноты (компоненты запаха) | Бергамот, Роза, Ваниль |
| `Perfume` | Сами парфюмы | "Citrus Dawn" от Maison Aurea, 2021 |
| `PerfumeFamily` | Связь парфюм↔семейство | Citrus Dawn относится к Цитрусовым и Свежим |
| `PerfumeNote` | Связь парфюм↔нота | Citrus Dawn содержит Бергамот, Грейпфрут, Мяту, Кедр |
| `WeatherRule` | Правило подбора | "hot, 25-50°C → Цитрусовые" |
| `Collection` | Избранные парфюмы юзера | admin сохранил Citrus Dawn |

**Связи между таблицами:**

- **1:M (один-ко-многим):**
  - Одна страна — много брендов
  - Один бренд — много парфюмов
  - Один пользователь — много записей в коллекции

- **M:M (многие-ко-многим):**
  - Один парфюм имеет много семейств, одно семейство содержит много парфюмов
  - Один парфюм имеет много нот, одна нота входит во много парфюмов

ER-диаграмма: `docs/ER.md` (Mermaid открывается прямо на GitHub).

## 7. Как работает главная фича — подбор по погоде

Это **изюминка проекта** и самое важное на защите.

### Алгоритм пошагово:

1. Юзер вводит "Москва" в форму
2. Сервер отправляет запрос на **Open-Meteo Geocoding API** → получает координаты (широту/долготу)
3. Сервер отправляет второй запрос на **Open-Meteo Forecast API** → получает текущую погоду:
   - температура
   - код погоды (число WMO: 0 = ясно, 71 = снег, и т.д.)
   - влажность
4. Функция `classify_condition` превращает код+температуру в одно из условий:
   - `hot` (≥25°C)
   - `warm` (15–24°C)
   - `cool` (5–14°C)
   - `cold` (<5°C)
   - `rain` (если идёт дождь, независимо от t)
   - `snow` (если идёт снег)
5. SQL-запрос ищет все `WeatherRule`, где это условие подходит и температура попадает в диапазон
6. Из правил вытаскиваются семейства парфюмов
7. Из БД выбираются парфюмы, относящиеся к этим семействам, сортируются по совпадению и показываются юзеру (топ-10)

### Где это в коде:

- **`weather/services.py`** — вся логика
  - `OpenMeteoClient` — общение с внешним API
  - `classify_condition()` — превращение кода в условие
  - `PerfumeRecommender` — финальный подбор
  - **Страховки сети:** IPv4-форс + DoH-fallback через 1.1.1.1 (работает на машинах с битым DNS)
- **`weather/views.py`** → `recommend_view` — связывает всё с веб-страницей
- **`templates/weather/recommend.html`** — отображение результата

### Пример демо для защиты:

| Город | Условие | Семейства | Парфюмы |
|-------|---------|-----------|---------|
| Москва летом (+27) | hot | Цитрусовые, Свежие | Citrus Dawn, Pacific Wave, Solar Mint |
| Якутск зимой (-30) | snow | Восточные | Oud Sahara, Winter Amber |
| Лондон осенью (+10, дождь) | rain | Древесные | Forest Pulse, Desert Wood |

## 9. Возможные вопросы препода и ответы

### "Почему Django, а не Flask?"
Django — "батарейки включены": готовые админка, авторизация, формы, ORM, миграции. Это сильно ускорило разработку.

### "Что такое ORM?"
Object-Relational Mapping. Вместо SQL пишем Python:
```python
Perfume.objects.filter(brand__name="Maison Aurea")
```
Django сам превращает это в `SELECT * FROM perfume JOIN brand ...`.

### "Что такое миграции?"
Файлы, которые описывают изменения структуры БД. Когда меняешь модель, команда `makemigrations` создаёт файл миграции, `migrate` его применяет. Так история БД хранится в коде.

### "Что такое M:M?"
Связь "многие-ко-многим". Реализуется промежуточной таблицей. Пример: `Perfume` ↔ `Family`. Django создаёт скрытую таблицу `catalog_perfume_families` с двумя FK.

### "Зачем `select_related` и `prefetch_related`?"
Оптимизация. Без них Django сделал бы по запросу на каждую карточку (проблема N+1). С ними — один JOIN-запрос или предзагрузка одним батчем.

### "Что такое CSRF-токен?"
Защита от подделки межсайтовых запросов. Django вставляет скрытое поле `csrfmiddlewaretoken` в форму. Если оно не совпадает с сессией — POST отклоняется.

### "Почему AbstractUser, а не AbstractBaseUser?"
`AbstractUser` уже содержит `username`, `email`, `password`, `is_staff`, `is_superuser`, группы и пермишены. Не нужно изобретать. Достаточно унаследоваться и при желании добавить поля.

### "Что хранится в .env и зачем?"
Секреты (`DJANGO_SECRET_KEY`, `DEBUG`). Файл не коммитится в Git. В репо лежит `.env.example` как шаблон.

### "Почему Open-Meteo, а не OpenWeather?"
Open-Meteo бесплатный, без регистрации и ключей. Хватает для учебного проекта. Стабильный, отдаёт JSON, есть geocoding "из коробки".

### "Где валидация форм?"
- На уровне модели: `MinValueValidator(1800)`, `MaxValueValidator(2100)` для года
- На уровне формы: `clean_year` в `PerfumeForm` запрещает год меньше 1900
- Уникальность: `UniqueConstraint(name, brand)` — нельзя 2 парфюма с одним именем в одном бренде

### "Как реализованы права?"
- **Анонимы:** видят каталог и подбор, не могут добавлять в коллекцию (редирект на login)
- **Авторизованные:** + могут сохранять в избранное, видят профиль
- **Staff (admin):** + создание/редактирование/удаление парфюмов
- Реализовано через `LoginRequiredMixin` + кастомный `StaffRequiredMixin` (на основе `UserPassesTestMixin`)

### "Сколько тестов?"
12 юнит-тестов. Покрывают:
- Маппинг погодных кодов в условия
- Клиент Open-Meteo (с моками `unittest.mock`)
- Алгоритм подбора `PerfumeRecommender` (3 сценария: жара, холод, пустой результат)

Запуск: `python manage.py test`.

### "Что такое CBV?"
Class-Based View — представление-класс вместо функции. Django даёт готовые: `ListView` (списки), `DetailView` (одна запись), `CreateView/UpdateView/DeleteView` (формы). Меньше кода, больше переиспользования.

### "Что такое venv и зачем?"
Виртуальное окружение Python — изолированная папка с библиотеками только для этого проекта. Не засоряет систему. Создаётся командой `python -m venv venv`, активируется `source venv/bin/activate`.

### "Что такое requirements.txt?"
Список нужных Python-библиотек с версиями. Команда `pip freeze > requirements.txt` сохраняет, `pip install -r requirements.txt` ставит.

### "Что внутри `manage.py`?"
Скрипт, через который Django получает команды: `runserver`, `migrate`, `createsuperuser`, `test`, `shell`. Сам — всего 20 строк, всё делает Django.

### "Что если порт 8000 занят?"
Запустить на другом: `python manage.py runserver 8080`. Любой свободный порт 1024-65535 подходит.

### "Что делать, если на машине не работает интернет / DNS?"
В `weather/services.py` есть **два уровня страховки**:

1. **IPv4-форс:** на хостах без IPv6-маршрута Python `requests` падал бы с `Network is unreachable`. Патч `urllib3.util.connection.allowed_gai_family` заставляет все запросы идти по IPv4.
2. **DoH-fallback:** если системный DNS сломан (`Errno -5: No address associated with hostname`), `socket.getaddrinfo` патчится так, что при сбое идёт прямой запрос к **Cloudflare 1.1.1.1** через DNS-over-HTTPS. Это IP, поэтому DNS для самого DoH-запроса не нужен.

Так фича работает даже на машинах с проблемной сетью.

### "Что такое `{% extends %}`, `{% block %}` в шаблонах?"
Наследование шаблонов. `base.html` — каркас с навбаром и футером. Другие шаблоны пишут `{% extends "base.html" %}` и заполняют только блок `{% block content %}{% endblock %}`.

### "Почему SQLite, а не PostgreSQL?"
SQLite — файловая БД, не требует установки сервера. Подходит для учебного проекта и разработки. Для прода Django легко переключается на Postgres правкой одной строки в `settings.py`.

### "Что такое admin-панель и кто её писал?"
`/admin/` — встроенная админка Django. Сам Django её сгенерировал на основе моделей. Я только настроил `list_display`, `list_filter`, `search_fields` в файлах `admin.py`.
